phases / detail view
P0

Foundation / закрытие пробелов

~ 8–10 недель // prerequisite
+
Самая важная фаза. Без языка программирования и базы веба весь дальнейший путь — карточный домик. Цель: научиться читать чужой код, писать автоматизацию своими руками (без ИИ), уверенно понимать, что происходит в HTTP-запросе.
01
Python с нуля до автоматизации
CS50P (Harvard, бесплатно) → Automate the Boring Stuff → easy-задачи на Codewars. Без ИИ. Цель: писать скрипты 100+ строк руками.
critical python automation
02
JavaScript — read-only level
Читать клиентский код, понимать XSS, prototype pollution, eval/innerHTML. ~2 недели легкого знакомства, без углубления.
js read
03
Сети / HTTP до уровня заголовков
TCP/IP, DNS, TLS-handshake, прокси. HTTP до Secure/HttpOnly/SameSite, CORS, CSP. Книга: Computer Networking (Kurose) — выборочно.
networks http tls
04
English ≥ B1
Без английского зарубежный рынок и половина свежих материалов недоступны. 30–60 мин/день: техподкасты (Risky Business), документация, конференции с субтитрами.
critical soft
05
Pet-проект: каркас
Заведи Flask/FastAPI приложение. Пока без безопасности — просто работающее API + простой фронт. Это будет твой сквозной проект на все 14 месяцев.
flagship flask
  • Пишешь без ИИ Python-скрипт 100+ строк, работающий с API/файлами/regex
  • Читаешь чужой Python/JS-код и понимаешь, что он делает
  • Объясняешь словами, что происходит при открытии URL (DNS → TLS → HTTP → render)
  • Понимаешь и можешь настроить базовые HTTP security headers
  • Английский ≥ B1, читаешь документацию без словаря
// deliverables
GitHub: devsecops-journey (pet-проект, MVP) + devsecops-blog (5–7 первых write-up'ов).
P1

DevOps base / Dev и Ops до Sec

~ 10–12 недель // prerequisite
+
Прежде чем добавлять Sec, должен быть надёжный Dev и Ops. Школа 21 даст часть, но останутся пробелы в Linux internals, K8s, облаках. Закрываем их.
01
Linux internals
Процессы, namespaces, cgroups (база Docker), systemd, права, SELinux/AppArmor, bash 50+ строк. Книги: TLCL (Shotts), How Linux Works (Ward).
linux bash
02
Git advanced
Rebase, cherry-pick, конфликты, pre-commit hooks (пригодятся для secrets-scan), workflow (trunk-based, GitFlow).
git hooks
03
Docker уверенно
Dockerfile best practices, multi-stage, compose, networks, volumes, устройство под капотом — это ключ к контейнерной безопасности.
docker compose
04
CI/CD: Actions + GitLab
GitHub Actions (главный) + GitLab CI (доминирует в РФ). Уметь написать пайплайн с нуля: lint → test → build → publish. Пока без security.
ci actions gitlab
05
IaC: Terraform basics
LocalStack для эмуляции AWS бесплатно. Providers, resources, state, modules. HCL-синтаксис.
terraform iac
06
AWS Free Tier
EC2, S3, IAM, VPC, RDS. Развернуть свой проект, понять Shared Responsibility. Параллельно — обзорно Yandex Cloud (РФ-рынок).
aws cloud
07
Kubernetes — базово
Minikube/kind. Pods, Deployments, Services, ConfigMaps, Secrets, Ingress, kubectl. Глубже — в фазе 4.
k8s kubectl
  • Pet-проект работает в Docker + docker-compose
  • Настроен CI в Actions: lint, test, build, push image
  • Проект развёрнут в локальном K8s с Ingress
  • Написан Terraform-модуль 50–100 строк
  • Что-то развёрнуто в AWS Free Tier через Terraform
// deliverables
Pet-проект на этапе: контейнеризация + CI + базовый деплой в K8s/AWS. Write-up'ы по Docker, K8s, Terraform, GitHub Actions.
P2

AppSec foundation / веб-безопасность

~ 8–10 недель // core skill
+
Теперь Sec. Цель — не «прочитал OWASP Top 10», а уметь объяснить каждую уязвимость на уровне HTTP-запроса и продемонстрировать.
01
OWASP Top 10:2025 + API Top 10
Актуальная редакция (2025), плюс отдельный API Security Top 10 — критичен для современных приложений.
owasp api
02
PortSwigger Web Security Academy
Золотой стандарт. Бесплатно. Цель: все Apprentice + бóльшая часть Practitioner. Каждая лаба → write-up в блог.
must labs
03
Burp Suite Community
Proxy, Repeater, Intruder, Decoder — как продолжение руки. Burp Pro — позже, в работе. Параллельно: ZAP бесплатно.
burp zap
04
OAuth 2.0 / OIDC / JWT — глубоко
Не «что это», а типичные уязвимости: alg=none, key confusion, refresh token leak, redirect_uri bypass. Очень частая тема на интервью.
interview jwt oauth
05
Crypto for engineers
Симметричное/асимметричное, hashes, padding, TLS-suites, типичные ошибки (ECB, MD5 для паролей, hardcoded keys). Crypto 101 (free book) + Cryptopals Set 1–3.
crypto tls
06
Threat Modeling: STRIDE
Книга Шостака. DFD на свой pet-проект → STRIDE-анализ → mitigations. Tool: OWASP Threat Dragon.
stride tm
07
Secure coding audit
Самостоятельный security-аудит pet-проекта: валидация ввода, секреты, логи, deps, CORS/CSP/HSTS. Найти и пофиксить.
audit review
  • Закрыты все Apprentice + бóльшая часть Practitioner на PortSwigger
  • Без подсказок объясняешь и демонстрируешь: SQLi, XSS (S/R/DOM), SSRF, IDOR, XXE, deserialization, JWT-атаки
  • Threat model pet-проекта оформлен и в репозитории
  • Pet-проект прошёл собственный security-аудит и закрыты все находки
// deliverables
30+ PortSwigger write-up'ов, threat model, отчёт о собственном аудите проекта.
P3

DevSecOps tooling / собираем pipeline

~ 10–12 недель // core skill
+
Это и есть DevSecOps в чистом виде. На выходе у тебя — pet-проект с полноценным security-pipeline из 6+ инструментов. Главное достижение фазы.
01
SAST: Semgrep + SonarQube
Semgrep как основной (де-факто стандарт). SonarQube CE — для понимания enterprise-подхода (популярен в РФ). Цель: 5–10 собственных Semgrep-правил в публичном репозитории.
portfolio semgrep sonar
02
SCA + SBOM
Trivy (универсал), Grype + Syft (SBOM), Dependency-Check. Snyk free как референс. Сгенерировать CycloneDX/SPDX SBOM для своего проекта.
trivy sbom
03
Secrets scanning
Gitleaks + TruffleHog. Pre-commit hook на pet-репу так, чтобы секреты физически не могли попасть в commit.
gitleaks precommit
04
DAST в CI
ZAP baseline + full scan, интеграция в CI. Понимать ограничения DAST — почему он находит не всё.
zap dast
05
IaC security
Checkov (универсал), tfsec/Trivy для Terraform, Kube-bench + Kubescape для K8s. Запустить на своих манифестах, починить.
checkov iac
06
Container security
Trivy image-scan, Docker Bench, distroless-образы, multi-stage для уменьшения attack surface, non-root users.
trivy distroless
07
Secrets management: Vault
HashiCorp Vault локально: KV, dynamic secrets, CI integration. Концептуально: AWS Secrets Manager, Yandex Lockbox.
vault secrets
08
FINAL: full DevSecOps pipeline
pre-commit (gitleaks) → SAST (Semgrep) → SCA (Trivy) → build → image-scan → IaC-scan (Checkov) → deploy → DAST (ZAP). Quality gate: падение на High/Critical.
flagship e2e
  • Pet-проект имеет полный DevSecOps-пайплайн с 6+ инструментами
  • 5+ собственных Semgrep-правил опубликованы в отдельном репозитории
  • SBOM проекта сгенерирован, опубликован, отслеживается
  • Vault интегрирован для управления секретами
  • Опубликован подробный write-up «как я построил DevSecOps-пайплайн с нуля»
// deliverables
Pet-проект v1.0 с полным security-pipeline. Это твой главный аргумент на собеседовании.
P4

Cloud · K8s Security · Supply Chain

~ 10–12 недель // specialization
+
Здесь ты становишься реально востребованным, а не просто «знаком с инструментами». Главные темы: K8s security всерьёз, cloud security и supply chain — самая горячая тема 2024–2026.
01
Kubernetes Security всерьёз
Pod Security Admission (PSA — учи именно её, PSP удалены в 1.25). NetworkPolicy, RBAC уверенно, Admission Controllers (OPA Gatekeeper + Kyverno), Falco runtime.
market-hot k8s psa
02
CKS certification
Certified Kubernetes Security Specialist. Топ-1 по соотношению полезность/узнаваемость для DevSecOps. Подготовка: KodeKloud / Killer.sh.
cert cks
03
AWS Security deep
IAM (роли, policies, assume-role, condition keys), KMS, GuardDuty, Security Hub, CloudTrail. Аудит своего аккаунта через Prowler / ScoutSuite.
aws iam
04
Yandex Cloud (RU)
IAM, Lockbox, Audit Trails. Концептуально — для интервью в РФ-компаниях, которые мигрируют с AWS.
ycloud ru
05
SLSA framework
Supply-chain Levels for Software Artifacts: уровни 1–4, что значат, как достичь. Pet-проект → SLSA L2 минимум.
trend-2026 slsa
06
Sigstore + cosign
Подписать свой образ через cosign, опубликовать в Rekor (transparency log). Verify-policy в admission controller.
trend-2026 sigstore
07
Supply chain attacks: case study
Разобрать и описать в блоге: SolarWinds, Codecov, event-stream, ua-parser-js, xz-utils backdoor (2024). Корень проблемы и как защититься.
cases history
08
GitHub Actions security
Pinning to SHA, OIDC вместо long-lived secrets, harden-runner, минимальные permissions для workflows.
actions oidc
09
Compliance overview
ISO 27001, SOC 2, PCI DSS, NIST SSDF — на уровне «знаю, что и зачем». Для РФ: ФСТЭК, ГОСТ Р 56939.
compliance ru
  • Pet-проект развёрнут в production-like K8s с PSA, NetworkPolicy, RBAC, Falco
  • Образы pet-проекта подписаны через cosign, SBOM публикуется в Rekor
  • Аудит AWS-аккаунта через Prowler пройден, проблемы исправлены
  • В блоге — 2–3 разбора реальных supply-chain атак
  • (опционально, но сильно усиливает) сдан CKS
// deliverables
Pet-проект v2.0: K8s production-grade + supply chain security. Сертификация CKS (если идёшь на неё) — серьёзный буст резюме.
P5

AppSec deep · RU specifics · CTF

~ 6–8 недель // differentiation
+
Здесь ты получаешь то, что отличает middle-кандидата от джуна: навык ручного code review, понимание реальных CVE и знание специфики РФ-рынка.
01
Manual code review
Уязвимые проекты с GitHub ("intentionally vulnerable"). Найти руками → сравнить с Semgrep. Это то, что middle делает на интервью.
interview review
02
Real CVE deep-dive
Выбрать 3–5 свежих CVE по своему стеку. Прочитать advisory + патч на GitHub, понять корень. Опционально: PoC по описанию.
cve patches
03
RU-инструменты обзорно
PT Application Inspector, Solar appScreener, CodeScoring, MaxPatrol, R-Vision. Не глубоко — на уровне «знаю, чем отличаются», для интервью в крупный РФ-бизнес.
ru market
04
Bug Bounty (опц., но buster резюме)
HackerOne, Bugcrowd, Standoff Bug Bounty (RU), BI.ZONE Bug Bounty. Один реальный найденный bug = огромный плюс на интервью.
bb portfolio
05
CTF / hands-on
HackTheBox (Web), TryHackMe (AppSec paths). RU: Standoff CTF, RuCTF, школа SPbCTF. Развивает offensive mindset, который сильно помогает в AppSec.
htb ctf
06
Сообщества и митапы
DevSecOps Talks (TG), AppSec Village, OWASP Russia chapter, OFFZONE, ZeroNights, PHDays. Networking — недооценённый источник офферов.
network community
  • Сделан manual code review минимум 3 уязвимых проектов с отчётом
  • В блоге — разбор 3–5 реальных CVE по своему стеку
  • Знаешь различия PT AI / Solar appScreener / CodeScoring
  • Прошёл хотя бы 10 web-машин на HackTheBox/THM
  • (опц.) Подал хотя бы 1 отчёт в bug bounty программу
// deliverables
Code-review портфолио: репозиторий с ручными аудитами 3+ проектов, CVE-разборы в блоге.
P6

Job hunt / выход на рынок

~ 4–6 недель // execution
+
Цель — конвертировать накопленное в оффер. Можно вести параллельно с поздними фазами, не ждать «полной готовности».
01
CV: RU + EN
Два резюме. Структура: summary, key skills (с уровнями), проекты (pet-проект — №1), сертификации, soft, языки. Не «знаком с» — а «использовал в проекте Х для Y».
cv bilingual
02
GitHub polish
README ко всему, описания, скриншоты, ссылки на CI-runs. Закрепить 4–6 топовых репозиториев на профиле.
github portfolio
03
STAR-истории
5–7 готовых историй из своих проектов: Situation–Task–Action–Result. Проговорить вслух до естественности.
interview soft
04
Mock interviews
3–5 пробных: pramp.com, interviewing.io, знакомые из IT. Найти и пофиксить свои слабые места до настоящих собеседований.
mock
05
RU target list
Positive Technologies, BI.ZONE, Kaspersky, Solar JSOC, Сбер (СберТех), Яндекс, VK, Авито, Тинькофф, Альфа, Ozon, Wildberries, X5, МТС, Ростелеком, Awillix, Cyberus, Bastion. Многие имеют стажировки.
ru applications
06
INTL: remote-friendly
weworkremotely.com, remoteok.com, LinkedIn (DevSecOps + remote). Требует EN ≥ B2 и сильное портфолио.
remote intl
  • Резюме на двух языках готово
  • GitHub-профиль причёсан, 4–6 топ-репо закреплены
  • 5–7 STAR-историй проговорены
  • 3–5 mock interviews пройдены
  • Подано минимум 30 откликов, есть приглашения на интервью
